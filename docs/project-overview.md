# Project Overview

## Project Name
Smart Campus Event Management System

## Course
IT368 Software Engineering

## Project Purpose
The Smart Campus Event Management System is designed to help students, faculty members, and administrators manage campus events in one centralized platform. The system supports event creation, event registration, resource assignment, notifications, feedback collection, and audit logging.

## Problem Statement
Campus events are sometimes managed through manual communication or disconnected tools. This can cause confusion, duplicated work, missed updates, and difficulty tracking participation. The system solves this problem by providing one organized platform where events can be created, managed, and monitored.

## Main Users
- Student: Can view events, register for events, cancel registration, and submit feedback.
- Faculty: Can create, edit, and cancel events.
- Administrator: Can assign resources, generate reports, and review audit logs.

## Development Approach
The team uses GitHub to manage the project codebase and documentation. Development is divided into feature branches so that each team member can work on a separate part of the system. Pull Requests are used before merging into the main branch to support review, feedback, and quality control.
